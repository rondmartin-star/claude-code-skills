# v4.1 Intelligent Parallelization Strategy

**Principle:** Only parallelize operations that genuinely benefit from concurrency

**Date:** 2026-02-14
**Approach:** Selective, value-driven parallelization

---

## Core Principle

**Not all skills need parallelization.** Add parallelization only where it provides real value:

1. **Independent operations** that can run concurrently
2. **Batch processing** of multiple similar items
3. **High-volume operations** where speed matters
4. **No dependencies** between parallel operations

**Don't parallelize:**
- Sequential workflows with dependencies
- Single-item operations
- Low-volume operations
- Operations where overhead > benefit
